# Data Directory

本目录用于存放比赛数据及其派生文件。真实数据默认不提交到 GitHub。

建议目录结构：

```text
data/
├── raw/          # 原始比赛数据，保持不改动
├── interim/      # 中间处理结果，例如切分后的 trial
└── processed/    # 特征矩阵、标准化后的数组等
```

## 数据管理原则

1. `raw/` 中只放原始数据，不直接覆盖或修改。
2. 所有预处理结果写入 `interim/` 或 `processed/`。
3. 数据文件不上传到 GitHub，避免泄露比赛数据或提交大文件。
4. 后续读取代码应通过配置文件指定数据路径，避免在代码中写死本机绝对路径。
